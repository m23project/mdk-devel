Serveur DHCP ISC

Si le serveur DHCP externe est un serveur DHCP ''ISC'', vous pouvez utiliser ce dernier à la place du serveur DHCP m23.
  1. Choisissez ''PXE'' dans ''Type de boot'' lorsque vous ajoutez un nouveau client m23 par l'interface web. Saisissez le nom de l'ordinateur à assigner au client m23 par le serveur DHCP dans ''Nom du poste client''.
  2. Désactivez le serveur DHCP de votre serveur m23: En éditant le fichier suivant /etc/default/isc-dhcp-server sur le serveur m23, commentez la ligne ci-dessous en ajoutant un ''#'' au début de celle-ci INTERFACES=... cette dernière deviendra #INTERFACES=... puis ajoutez une nouvelle ligne INTERFACES='''' en dessous de celle précédemment commenté. Finalement, redémarrez le serveur DHCP en tapant la commande /etc/init.d/isc-dhcp-server restart Le serveur DHCP sera toujours fonctionnel mais ne répondra plus à aucune requète DHCP.
  3. Dans la configuration de votre serveur DHCP, activez l'option use-host-decl-names on pour le groupe contenant les clients m23.
  4. Mettez régulièrement à jour le groupe m23 dans la configuration de votre serveur DHCP (par exemple en utilisant une tâche ''cron'' ou une autre méthode automatisée) utilisant les données de votre serveur DHCP m23 par l'URL http://m23-Server-IP/exportDHCPSettings.php Cette URL fournit la configuration des clients m23, devant booter sur le réseau, dans le format attendu par le ''serveur DHCP ISC''. Le groupe pour les clients m23 pourrait par exemple ressembler au code ci-dessous. 
    group {
use-host-decl-names on; 
$\langle$A partir des donn\'ees de exportDHCPSettings.php ins\'er\'ees$\rangle$
}
