Renseignement concernant l'architecture du processeur

m23 est compatible avec les processeurs 32 et 64 bits. Une version 32 bits peut être installée sur un ordinateur 64 bits, l'inverse n'est pas possible. Pour profiter de l'entière potentialité d'un microprocesseur 64 bits, vous devez choisir amd64. Cette catégorie amd64 ne comportent pas uniquement les processeurs AMD, mais également ceux d'Intel. Par exemple, les processeurs Intel Pentium D, Pentium Extreme Edition, Celeron D, Core 2 et les processeurs AMD Sempron, Athlon 64, Athlon X2 et Phenom en font partie.
Adresse MAC: Adresse MAC de la carte réseau du poste client (par ex. 00D0B723865C)
Adresse IP: Adresse IP fixe du poste client (par ex. 192.168.1.23)
Masque réseau: Masque de sous-réseau (par ex. 255.255.255.0)
Passerelle: Adresse IP de la passerelle (gateway) pour les connexions au réseau internet ou à d'autres connexions externes à votre propre réseau
DNS1: Adresse IP du serveur DNS de résolution des noms internet
DNS2 (facultatif): Adresse IP du serveur DNS backup
Serveur proxy des paquets: Le poste client essaie de télécharger les paquets logiciels sur l'adresse IP du serveur proxy indiqué dans ce champ. En général, il s'agit de l'adresse IP du serveur m23, mais il est également possible d'indiquer un autre serveur proxy. Si vous laissez ce champs de saisie vide, le client téléchargera les paquets logiciels sur internet. Entrez également le numéro du port du serveur proxy. Dans le cas du serveur m23, c'est le port 2323. Lorsque vous ouvrez la boîte de dialogue Ajouter un client, le serveur m23 est le serveur proxy par défaut.
Groupe (facultatif): Nom du groupe de postes client auquel appartient ce poste client
Mot de passe de l&apos;utilisateur: Mot de passe de l'utilisateur employé à la première connexion au système. Ce mot de passe comporte six chiffres sélectionnés au hasard, vous pouvez également choisir un mot de passe de votre choix.
Mot de passe du superutilisateur root: Mot de passe employé pour l'accès au poste client en tant que super-utilisateur (root) . Il comporte huit chiffres sélectionnés au hasard, vous pouvez également choisir un mot de passe de votre choix.
Fuseau horaire: Définition du fuseau horaire d'appartenance du poste client.
Déterminer l&apos;heure système par NTP: Option de synchronisation de l'heure du poste client par un serveur de temps NTP accessible sur internet.
Installer les pilotes de l&apos;imprimante puis détecter l&apos;imprimante: Détection puis installation automatique des imprimantes et de leurs pilotes.
Enregistrer les données d&apos;ouverture de session locales sur le poste client.: Cochez cette option si vous désirez un compte local sur le poste client pour l'utilisateur.
Ne pas utiliser le LDAP: Pour utiliser uniquement l'authentification locale sur les postes client, cochez Ne pas utiliser le LDAP.
Lire les données d&apos;ouverture de session depuis le serveur LDAP sélectionné.: / Lecture des données d'identification UCS: Pour utiliser les données d'authentification du compte utilisateur enregistrées sur un serveur LDAP ou sur un serveur UCS, cliquez respectivement sur Lire les données d&apos;ouverture de session depuis le serveur LDAP sélectionné. ou ''Lecture des données d'identification UCS''..
Enregistrer les données d&apos;ouverture de session sur le serveur LDAP. / Sauvegarde des données d'identification sur UCS: Enregistrement des données de l' utilisateur renseignées ci-dessus, dans la base de données du serveur LDAP m23 ou le serveur UCS, pour l'authentification sur les postes client, cochez Enregistrer les données d&apos;ouverture de session sur le serveur LDAP. ou respectivement ''Sauvegarde des données d'identification sur UCS''.. Le bon fonctionnement de cette option nécessite un accès complet au serveur LDAP pour l'enregistrement des données sur celui-ci.
ID de l&apos;utilisateur, ID du groupe: De plus, vous devez renseigner l'ID de l&apos;utilisateur et l'ID du groupe pour le compte utilisateur. Ceci est très important; surtout lors de l'utilisation de NFS pour les répertoires personnels.
Nom du serveur LDAP: Sélectionnez un serveur LDAP dans la liste Nom du serveur LDAP. Si le serveur LDAP souhaité n'y apparaissait pas, vous pouvez l'ajouter en cliquant sur Administrer le serveur LDAP. Si m23 a été installé sur un serveur UCS, cette dernière option est invalide. Le service LDAP du serveur UCS sera utilisé à la place.
Répertoire personnel sur le serveur NFS: Ce champ assure l'enregistrement des données du répertoire personnel de l'utilisateur sur un serveur NFS central, afin qu'elles puissent être accédées de tout poste client (en combination avec LDAP). Pour activer le NFS, entrez le nom de l'ordinateur ou son adresse IP et le chemin pour l'enregistrement des répertoires personnels, comme par exemple: 
192.168.1.23:/nfs-homes
. Si m23 fonctionne sur un serveur UCS, cette dernière option est invalide.
Validez l'ajout du poste client en cliquant sur Definir.
